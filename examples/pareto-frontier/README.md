# Pareto frontier example

This is a compact code sample from a larger narrated explainer. It demonstrates:

- typed multidimensional data
- a tested-in-code dominance rule
- separate two-dimensional and six-dimensional Pareto sets
- persistent points that gain color, area, and frontier rings
- a mostly monochrome Excalidraw-style visual language

Install, copy the locally packaged Excalifont asset, and render:

```bash
npm install
npm run setup:font
npm run typecheck
npm run render
```

The example intentionally ships without narration. Add only audio you own or are licensed to use. If adding an `<Audio>` element, reference a neutral filename such as `public/voiceover.wav`; keep provider request payloads and voice metadata outside the repository.

The fixture is a dated public benchmark snapshot used to reproduce the technique. Refresh and cite the data before presenting it as current.
